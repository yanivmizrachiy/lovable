# PROTOCOL

## עקרונות עבודה
- שינויי תוכן (מלל/ניסוח/נוסחאות) מבוצעים רק לפי טקסט שהבעלים מספק. אין עריכה לשונית אוטומטית.
- כל שינוי טכני חייב לעדכן גם את RULES.md באותו סט שינוי.
- עבודה אינקרמנטלית: קבוצה קטנה של עמודים, ואז QA חוסם.

## זרימת עבודה מומלצת
1) הבעלים מספק קובץ מקור (PDF) או טקסט מאושר.
2) ניתוח מקור לפי פרק 13 ב-RULES.md ותיעוד בכל עמוד.
3) יצירת עמוד HTML לכל עמוד מקור + קובץ docs תואם.
4) עדכון אינדקס הספר ב-RULES.md.
5) הרצה: `npm run qa`.
6) הפקת PDF: `npm run pdf` (לפי אינדקס הספר שב-RULES.md).

## כללים תפעוליים
- אין יצירת עמודים ריקים בריפו ללא מקור מאושר.
- אין שימוש במספור מספרי לשאלות; רק בולטים לפי RULES.md.
- כל גרף/שרטוט חייב להיות SVG אלא אם יש נימוק מתועד.
